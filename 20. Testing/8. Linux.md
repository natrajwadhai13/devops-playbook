# 🐧 Linux Commands for Software Testing (Interview Ready)

## 📂 File & Directory Commands

| Command                           | Description                   | Example          |
| --------------------------------- | ----------------------------- | ---------------- |
| `pwd`                             | Show current directory/folder | `pwd`            |
| `ls`                              | List files                    | `ls -l`          |
| `cd`                              | Change directory              | `cd /home/user`  |
| `mkdir`                           | Create directory              | `mkdir test`     |
| `rmdir`                           | Remove empty directory        | `rmdir test`     |
| `rm`or -rf for file and directory | Delete file/directory         | `rm -rf test`    |
| `cp`                              | Copy files                    | `cp file1 file2` |
| `mv`                              | Move/rename file              | `mv file1 file2` |
| `touch`                           | Create empty file             | `touch test.txt` |

## 📄 File Viewing Commands

| Command   | Description              | Example              |
| --------- | ------------------------ | -------------------- |
| `cat`     | View file content        | `cat file.txt`       |
| `less`    | View large file (scroll) | `less log.txt`       |
| `more`    | View file page by page   | `more file.txt`      |
| `head`    | First 10 lines           | `head -n 5 file.txt` |
| `tail`    | Last 10 lines            | `tail -n 5 file.txt` |
| `tail -f` | Live log monitoring      | `tail -f app.log`    |

👉 **Interview Tip:** `tail -f` is very important for **log monitoring in testing**.

## 🔍 Search & Filter Commands

| Command   | Description             | Example                   |
| --------- | ----------------------- | ------------------------- |
| `grep`    | Search text in file     | `grep "error" log.txt`    |
| `grep -i` | Case insensitive search | `grep -i error log.txt`   |
| `find`    | Find files              | `find / -name "test.txt"` |
| `locate`  | Quick file search       | `locate test.txt`         |
| `wc`      | Count lines/words       | `wc -l file.txt`          |
| `sort`    | Sort file content       | `sort file.txt`           |
| `uniq`    | Remove duplicates       | `uniq file.txt`           |

👉 **Interview Tip:** `grep + tail` combination is commonly asked.

## ⚙️ Permissions & Ownership

| Command | Description        | Example               |
| ------- | ------------------ | --------------------- |
| `chmod` | Change permissions | `chmod 755 file.sh`   |
| `chown` | Change owner       | `chown user file.txt` |
| `chgrp` | Change group       | `chgrp dev file.txt`  |

👉 Example:

```
chmod 777 file.txt
```

# 🎯 QA Tester Linux Interview Questions & Answers (Basic)

## 1. What is Linux?

**Answer:** Linux is an open-source operating system used to manage hardware and run applications. In testing, it is commonly used for servers, logs, and deployments.

---

## 2. What is the difference between `ls` and `ls -l`?

**Answer:**

- `ls` → shows file names
- `ls -l` → shows detailed info (permissions, owner, size, date)

---

## 3. How do you check current directory?

**Answer:**

```
pwd
```

---

## 4. How do you view logs in real time?

**Answer:**

```
tail -f app.log
```

👉 Used to monitor application logs during testing.

---

## 5. How do you search "ERROR" in logs?

**Answer:**

```
grep ERROR app.log
```

---

## 6. Combine tail and grep?

**Answer:**

```
tail -f app.log | grep ERROR
```

👉 Very commonly asked in interviews.

---

## 7. How do you check file content?

**Answer:**

```
cat file.txt
less file.txt
```

---

## 8. Difference between `cat` and `less`?

**Answer:**

- `cat` → shows full file
- `less` → scrollable (used for large files)

---

## 9. How do you check disk space?

**Answer:**

```
df -h
```

---

## 10. How do you check folder size?

**Answer:**

```
du -sh folder/
```

---

## 11. How do you check running processes?

**Answer:**

```
ps -ef
```

---

## 12. How do you kill a process?

**Answer:**

```
kill -9 PID
```

---

## 13. What is PID?

**Answer:** Process ID (unique ID for each running process).

---

## 14. How do you check server connectivity?

**Answer:**

```
ping google.com
```

---

## 15. How do you test an API using Linux?

**Answer:**

```
curl http://api-url
```

👉 Important for API testing.

---

## 16. What is `chmod`?

**Answer:** Used to change file permissions.

Example:

```
chmod 755 file.sh
```

---

## 17. What is 777 permission?

**Answer:**

- Read, write, execute for all users (not secure in production)

---

## 18. How do you create a file?

**Answer:**

```
touch file.txt
```

---

## 19. How do you delete a file?

**Answer:**

```
rm file.txt
```

---

## 20. What is `find` command?

**Answer:** Used to search files in system.

Example:

```
find / -name "test.txt"
```

---

## 21. What is `grep` used for?

**Answer:** Used to search text inside files.

---

## 22. How do you check memory usage?

**Answer:**

```
free -m
```

---

## 23. What is `top` command?

**Answer:** Shows real-time CPU and memory usage.

---

## 24. What is log file in testing?

**Answer:** File that stores application events, errors, and debug info.

---

## 25. Real-time scenario: App not working, what will you do?

**Answer:**

1. Check logs → `tail -f app.log`
2. Search errors → `grep ERROR`
3. Check process → `ps -ef`
4. Check server → `ping`
5. Check disk → `df -h`

---

## 26. Difference between `rm` and `rmdir`?

**Answer:**

- `rm` → delete files/directories
- `rmdir` → delete empty directory only

---

## 27. What is `history` command?

**Answer:** Shows previously used commands.

---

## 28. What is `whoami`?

**Answer:** Shows current logged-in user.

---

## 29. What is environment in testing?

**Answer:** Setup where application runs (Dev, QA, UAT, Prod).

---

## 30. Why Linux is important for QA?

**Answer:**

- Most servers run on Linux
- Used for log analysis
- API testing
- Deployment validation

---

## ✅ Final Tip for Interview

- Don’t just say command → explain **real-time usage**
- Focus on: `tail`, `grep`, `curl`, `ps`, `df`
- Give **practical examples**
