## **1. What is Docker?**

**Docker** is a **container platform** used to:

* Package applications with dependencies
* Run isolated lightweight containers
* Ensure consistency across environments

---

## **2. What is Kubernetes?**

* **Kubernetes (K8s)** is a **container orchestration platform** that manages containerized applications across multiple hosts.

* Kubernetes solves the limitations of Docker.

# **5. Kubernetes Architecture**

Kubernetes architecture is divided into: **A) Control Plane (Master Node)** and **B) Data Plane (Worker Node)**



Below is the high-level Kubernetes Cluster Architecture:

![Kubernetes Architecture](https://raw.githubusercontent.com/natrajwadhai13/devops-playbook/main/images/kubernetes-cluster-architecture.svg)



| **Section**                        | **Component**                      | **Description / Responsibilities**                                                                                              |
| ---------------------------------- | ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **A) Control Plane (Master Node)** | **API Server**                     | ‚Ä¢ Main entry point for all Kubernetes requests<br>‚Ä¢ Handles kubectl commands<br>‚Ä¢ Validates and processes API requests          |
|                                    | **Scheduler**                      | ‚Ä¢ Assigns pods to nodes<br>‚Ä¢ Decides *which node* runs a Pod<br>‚Ä¢ Scheduling based on CPU, memory, affinity, taints/tolerations                            |
|                                    | **etcd**                           | ‚Ä¢ Distributed key-value store<br>‚Ä¢ Stores entire cluster state (desired + actual)                                               |
|                                    | **Controller Manager**             | ‚Ä¢ Runs background controllers<br>‚Ä¢ Ensures correct number of replicas<br>‚Ä¢ Controllers: Deployment, ReplicaSet, Node Controller |
|                                    | **Cloud Controller Manager (CCM)** | ‚Ä¢ Integrates Kubernetes with cloud providers (AWS, Azure, GCP)<br>‚Ä¢ Manages load balancers, node lifecycle, cloud networking    |
| **B) Data Plane (Worker Node)**    | **Kubelet**                        | ‚Ä¢ Ensures containers are running in Pods<br>‚Ä¢ Communicates with API Server<br>‚Ä¢ Reports node and pod status                     |
|                                    | **Kube-Proxy**                     | ‚Ä¢ Handles networking and load balancing<br>‚Ä¢ Manages cluster networking rules<br>‚Ä¢ Implements iptables/ipvs rules<br>‚Ä¢ Handles service-level load balancing                |
|                                    | **Container Runtime**              | ‚Ä¢ Runs containers on the node<br>‚Ä¢ Examples: containerd, CRI-O, Docker (via containerd)                                         |

# **6. Kubernetes Key Features**

### ‚úî **Cluster**

Group of nodes (master + workers).

### ‚úî **Auto-Healing**

Restart Pods automatically.

### ‚úî **Auto-Scaling**

Replicas change dynamically.

### ‚úî **Enterprise Support**

Security, load balancing, storage, secrets management.
