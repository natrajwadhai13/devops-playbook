## **1. What is Docker?**

**Docker** is a **container platform** used to:

- Package applications with dependencies
- Run isolated lightweight containers
- Ensure consistency across environments

---

## **2. What is Kubernetes?**

- **Kubernetes (K8s)** is a **container orchestration platform** that manages containerized applications across multiple hosts.

- Kubernetes solves the limitations of Docker.

---

## ✅ Docker vs Kubernetes – Comparison Table

| **Topic**                             | **Docker**                                                                                                                                                                      | **Kubernetes**                                                                                                                                                                                                                              |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **One-Line Summary**                  | A **container platform** to build and run containers.                                                                                                                           | An **orchestration platform** to manage containers at scale with auto-healing, auto-scaling, load balancing & enterprise features.                                                                                                          |
| **Problem 1: Single Host Limitation** | _ Runs containers on **one host** only. <br>_ If one container uses extra memory/CPU → other containers are affected. <br>\* If host fails → all containers stop.               | **Multi-Host Cluster** <br>Kubernetes uses a **cluster** of: <br>• **Control Plane (Master)** <br>• **Worker Nodes** <br>Workloads are distributed across many servers safely.                                                              |
| **Problem 2: No Auto-Healing**        | _ If a container crashes, Docker **does not restart it automatically**. <br>_ DevOps must manually restart & monitor.                                                           | **Auto-Healing** <br>_ If a Pod fails → Kubernetes automatically creates a new one. <br>_ Managed by **ReplicaSet, Deployments**.                                                                                                           |
| **Problem 3: No Auto-Scaling**        | _ Cannot auto-increase container count during high load. <br>_ No built-in load balancing.                                                                                      | **Auto-Scaling (HPA/VPA)** <br>_ Scale from **1 → 10 pods** automatically. <br>_ Based on **CPU, Memory, Traffic, Custom Metrics**.                                                                                                         |
| **Problem 4: Not Enterprise-Ready**   | Docker alone does **not** provide: <br>• Load Balancers <br>• Auto Scaling <br>• Auto Healing <br>• Firewall/Network policies <br>• API Gateway <br>• Enterprise-grade security | **Enterprise-Ready Orchestration** <br>Kubernetes provides: <br>• Load Balancing <br>• Service Discovery <br>• Network Policies (Firewall) <br>• RBAC Security <br>• Rolling Updates <br>• API Gateway Integration <br>• Secrets Management |
| **Architecture**                      | Single-engine container runtime.                                                                                                                                                | **Cluster Architecture**: <br>**Control Plane** – API Server, etcd, Scheduler, Controllers <br>**Worker Nodes** – Kubelet, Kube-Proxy, Container Runtime                                                                                    |
| **Self-Healing**                      | ❌ Not available                                                                                                                                                                | ✔ **Auto-Healing Pods**                                                                                                                                                                                                                     |
| **Scaling**                           | ❌ Manual scaling                                                                                                                                                               | ✔ **HPA, VPA, Cluster Autoscaling**                                                                                                                                                                                                         |
| **Networking**                        | Basic bridge networking                                                                                                                                                         | ✔ **Services, Ingress, Load Balancing**                                                                                                                                                                                                     |
| **Security**                          | Minimal                                                                                                                                                                         | ✔ RBAC, Secrets, PSS, Network Policies                                                                                                                                                                                                      |
| **Use Cases**                         | Local development, small setups                                                                                                                                                 | Production, enterprise, microservices, multi-node clusters                                                                                                                                                                                  |

# **5. Kubernetes Architecture**

Kubernetes architecture is divided into: **A) Control Plane (Master Node)** and **B) Data Plane (Worker Node)**

Below is the high-level Kubernetes Cluster Architecture:

![Kubernetes Architecture](https://raw.githubusercontent.com/natrajwadhai13/devops-playbook/main/images/kubernetes-cluster-architecture.svg)

| **Section**                        | **Component**                      | **Description / Responsibilities**                                                                                                                          |
| ---------------------------------- | ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A) Control Plane (Master Node)** | **API Server**                     | • Main entry point for all Kubernetes requests<br>• Handles kubectl commands<br>• Validates and processes API requests                                      |
|                                    | **Scheduler**                      | • Assigns pods to nodes<br>• Decides _which node_ runs a Pod<br>• Scheduling based on CPU, memory, affinity, taints/tolerations                             |
|                                    | **etcd**                           | • Distributed key-value store<br>• Stores entire cluster state (desired + actual)                                                                           |
|                                    | **Controller Manager**             | • Runs background controllers<br>• Ensures correct number of replicas<br>• Controllers: Deployment, ReplicaSet, Node Controller                             |
|                                    | **Cloud Controller Manager (CCM)** | • Integrates Kubernetes with cloud providers (AWS, Azure, GCP)<br>• Manages load balancers, node lifecycle, cloud networking                                |
| **B) Data Plane (Worker Node)**    | **Kubelet**                        | • Ensures containers are running in Pods<br>• Communicates with API Server<br>• Reports node and pod status                                                 |
|                                    | **Kube-Proxy**                     | • Handles networking and load balancing<br>• Manages cluster networking rules<br>• Implements iptables/ipvs rules<br>• Handles service-level load balancing |
|                                    | **Container Runtime**              | • Runs containers on the node<br>• Examples: containerd, CRI-O, Docker (via containerd)                                                                     |

# **6. Kubernetes Key Features**

### ✔ **Cluster**

Group of nodes (master + workers).

### ✔ **Auto-Healing**

Restart Pods automatically.

### ✔ **Auto-Scaling**

Replicas change dynamically.

### ✔ **Enterprise Support**

Security, load balancing, storage, secrets management.
