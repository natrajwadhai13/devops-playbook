# **1. What is Docker?**

**Docker** is a **container platform** used to:

* Package applications with dependencies
* Run isolated lightweight containers
* Ensure consistency across environments

---

# **2. What is Kubernetes?**

**Kubernetes (K8s)** is a **container orchestration platform** that manages containerized applications across multiple hosts.

Kubernetes solves the limitations of Docker.

---


| Topic | Docker  | Kubernetes |
|--------|-------------|-------------|
| **One-Line Summary** | container platform. | orchestration platform to manage containers at scale with auto-healing, auto-scaling, load balancing, and enterprise features.|
| **Problem 1** | **Single-Host Limitation** * Docker runs containers on **one host** only.
* If one container consumes too much memory/CPU → other containers on the same machine can be affected.
* If host fails → all containers die. |**Multi-Host Cluster** Kubernetes forms a **cluster** of:

* **Master Nodes (Control Plane)**
* **Worker Nodes (Data Plane)**

Runs workloads across many servers safely. |
| **Problem 2** | **No Auto-Healing** * If a container crashes, Docker **does not restart it automatically**.
* DevOps team must check and restart containers manually. | **Auto-Healing** * If a Pod fails → Kubernetes automatically creates a new one.
* Ensured by controllers like **ReplicaSet**. |
| **Problem 3** | **No Auto-Scaling** * Docker cannot automatically increase container count during high load.
* No internal load balancer to distribute traffic.
* Enterprises need scaling (1 → 10 containers automatically). | ------------- |

* Load balancers
* Auto-scaling
* Auto-healing

### ✔ **Auto-Healing**

* If a Pod fails → Kubernetes automatically creates a new one.
* Ensured by controllers like **ReplicaSet**.

---

### ✔ **Auto-Scaling**

* Kubernetes Horizontal Pod Autoscaler (HPA)
* Can scale from **1 → 10 pods** based on:

  * CPU
  * Memory
  * Custom metrics (traffic)

---

### ✔ **Enterprise-Grade Orchestration**

K8s provides:

* Load balancing
* Service discovery
* RBAC security
* Firewall (NetworkPolicies)
* Rolling updates
* API gateway integration
* Secrets management


---

# **5. Kubernetes Architecture**

Kubernetes architecture is divided into:

---

## **A) Control Plane (Master Node)**

Responsible for cluster management.

### **Components:**

### **1. API Server**

* Entry point for external/internal requests
* kubectl commands go to API server
* Validates and processes requests

---

### **2. Scheduler**

* Decides *which node* will run the Pod
* Based on:

  * CPU
  * Memory
  * Taints/Tolerations
  * Node Affinity

---

### **3. etcd**

* Key-value database
* Stores entire cluster state (desired + actual state)

---

### **4. Controller Manager**

Runs controllers such as:

* ReplicaSet
* Deployment
* Node controller

Ensures:

* Pods always running
* Required number of replicas present

---

### **5. Cloud Controller Manager (CCM)**

Integrates with cloud:

* AWS EKS
* Azure AKS
* GCP GKE

Handles:

* Load balancers
* Node lifecycles
* Cloud-specific networking

---

# **B) Data Plane (Worker Node)**

Where actual applications run.

### **Components:**

### **1. Kubelet**

* Pod creation
* Ensures containers are running
* Reports node health to control plane

---

### **2. Kube-Proxy**

* Manages networking rules
* iptables / ipvs routing
* Service load balancing

---

### **3. Container Runtime**

Used to run containers:

* containerd
* CRI-O
* Docker (deprecated in K8s but still used via containerd)

---

# **6. Kubernetes Key Features**

### ✔ **Cluster**

Group of nodes (master + workers).

### ✔ **Auto-Healing**

Restart Pods automatically.

### ✔ **Auto-Scaling**

Replicas change dynamically.

### ✔ **Enterprise Support**

Security, load balancing, storage, secrets management.
