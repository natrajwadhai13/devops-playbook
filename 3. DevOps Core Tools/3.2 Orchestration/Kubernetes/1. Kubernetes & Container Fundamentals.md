# **1. What is Docker?**

**Docker** is a **container platform** used to:

* Package applications with dependencies
* Run isolated lightweight containers
* Ensure consistency across environments

---

# **2. What is Kubernetes?**

**Kubernetes (K8s)** is a **container orchestration platform** that manages containerized applications across multiple hosts.

Kubernetes solves the limitations of Docker.

---

| **Topic**                             | **Docker**                                                                                                                                                                      | **Kubernetes**                                                                                                                                                                                                                              |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **One-Line Summary**                  | A **container platform** to build and run containers.                                                                                                                           | An **orchestration platform** to manage containers at scale with auto-healing, auto-scaling, load balancing & enterprise features.                                                                                                          |
| **Problem 1: Single Host Limitation** | * Runs containers on **one host** only. <br>* If one container uses extra memory/CPU → other containers are affected. <br>* If host fails → all containers stop.                | **Multi-Host Cluster** <br>Kubernetes uses a **cluster** of: <br>• **Control Plane (Master)** <br>• **Worker Nodes** <br>Workloads are distributed across many servers safely.                                                              |
| **Problem 2: No Auto-Healing**        | * If a container crashes, Docker **does not restart it automatically**. <br>* DevOps must manually restart & monitor.                                                           | **Auto-Healing** <br>* If a Pod fails → Kubernetes automatically creates a new one. <br>* Managed by **ReplicaSet, Deployments**.                                                                                                           |
| **Problem 3: No Auto-Scaling**        | * Cannot auto-increase container count during high load. <br>* No built-in load balancing.                                                                                      | **Auto-Scaling (HPA/VPA)** <br>* Scale from **1 → 10 pods** automatically. <br>* Based on **CPU, Memory, Traffic, Custom Metrics**.                                                                                                         |
| **Problem 4: Not Enterprise-Ready**   | Docker alone does **not** provide: <br>• Load Balancers <br>• Auto Scaling <br>• Auto Healing <br>• Firewall/Network policies <br>• API Gateway <br>• Enterprise-grade security | **Enterprise-Ready Orchestration** <br>Kubernetes provides: <br>• Load Balancing <br>• Service Discovery <br>• Network Policies (Firewall) <br>• RBAC Security <br>• Rolling Updates <br>• API Gateway Integration <br>• Secrets Management |
| **Architecture**                      | Single-engine container runtime.                                                                                                                                                | **Cluster Architecture**: <br>**Control Plane** – API Server, etcd, Scheduler, Controllers <br>**Worker Nodes** – Kubelet, Kube-Proxy, Container Runtime                                                                                    |
| **Self-Healing**                      | ❌ Not available                                                                                                                                                                 | ✔ **Auto-Healing Pods**                                                                                                                                                                                                                     |
| **Scaling**                           | ❌ Manual scaling                                                                                                                                                                | ✔ **HPA, VPA, Cluster Autoscaling**                                                                                                                                                                                                         |
| **Networking**                        | Basic bridge networking                                                                                                                                                         | ✔ **Services, Ingress, Load Balancing**                                                                                                                                                                                                     |
| **Security**                          | Minimal                                                                                                                                                                         | ✔ RBAC, Secrets, PSS, Network Policies                                                                                                                                                                                                      |
| **Use Cases**                         | Local development, small setups                                                                                                                                                 | Production, enterprise, microservices, multi-node clusters                                                                                                                                                                                  |

# **5. Kubernetes Architecture**

Kubernetes architecture is divided into:

---

## **A) Control Plane (Master Node)**

Responsible for cluster management.

### **Components:**

### **1. API Server**

* Entry point for external/internal requests
* kubectl commands go to API server
* Validates and processes requests

---

### **2. Scheduler**

* Decides *which node* will run the Pod
* Based on:

  * CPU
  * Memory
  * Taints/Tolerations
  * Node Affinity

---

### **3. etcd**

* Key-value database
* Stores entire cluster state (desired + actual state)

---

### **4. Controller Manager**

Runs controllers such as:

* ReplicaSet
* Deployment
* Node controller

Ensures:

* Pods always running
* Required number of replicas present

---

### **5. Cloud Controller Manager (CCM)**

Integrates with cloud:

* AWS EKS
* Azure AKS
* GCP GKE

Handles:

* Load balancers
* Node lifecycles
* Cloud-specific networking

---

# **B) Data Plane (Worker Node)**

Where actual applications run.

### **Components:**

### **1. Kubelet**

* Pod creation
* Ensures containers are running
* Reports node health to control plane

---

### **2. Kube-Proxy**

* Manages networking rules
* iptables / ipvs routing
* Service load balancing

---

### **3. Container Runtime**

Used to run containers:

* containerd
* CRI-O
* Docker (deprecated in K8s but still used via containerd)

---

# **6. Kubernetes Key Features**

### ✔ **Cluster**

Group of nodes (master + workers).

### ✔ **Auto-Healing**

Restart Pods automatically.

### ✔ **Auto-Scaling**

Replicas change dynamically.

### ✔ **Enterprise Support**

Security, load balancing, storage, secrets management.
