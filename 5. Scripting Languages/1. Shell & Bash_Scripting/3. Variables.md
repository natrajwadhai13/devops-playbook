# Module 3: Variables

Variables are used to **store data** (text, numbers, paths, command output, etc.) that can be reused throughout a shell script.

---

# What is a Variable?

A **variable** is a named storage location that holds a value.

### Syntax

```bash
variable_name=value
```

> **Important:** Do **not** use spaces around `=`.

✅ Correct

```bash
name=Natraj
age=30
city=Pune
```

❌ Wrong

```bash
name = Natraj
```

---

# Rules for Variable Names

- Can contain letters, numbers, and `_`
- Cannot start with a number
- Case-sensitive (`NAME` and `name` are different)
- No spaces or special characters (except `_`)

✅ Valid

```bash
name=Natraj
AGE=30
user_name=admin
```

❌ Invalid

```bash
1name=test
user-name=test
my name=test
```

---

# Accessing Variables

Use `$` before the variable name.

```bash
name=Natraj

echo $name
```

Output

```text
Natraj
```

---

# User Variables

Variables created by the user inside a shell or script.

Example

```bash
#!/bin/bash

name="Natraj"
company="Infosys"
experience=7

echo "Name : $name"
echo "Company : $company"
echo "Experience : $experience Years"
```

Output

```text
Name : Natraj
Company : Infosys
Experience : 7 Years
```

---

# Environment Variables

Environment variables are predefined system variables available to all programs and child processes.

Display all environment variables

```bash
printenv
```

or

```bash
env
```

---

## Common Environment Variables

| Variable    | Description               |
| ----------- | ------------------------- |
| `$HOME`     | User Home Directory       |
| `$USER`     | Current Username          |
| `$PWD`      | Present Working Directory |
| `$PATH`     | Command Search Path       |
| `$SHELL`    | Current Shell             |
| `$HOSTNAME` | System Hostname           |
| `$LOGNAME`  | Login Name                |
| `$LANG`     | System Language           |
| `$TERM`     | Terminal Type             |
| `$OLDPWD`   | Previous Directory        |

Example

```bash
echo $HOME
echo $USER
echo $PWD
echo $PATH
echo $SHELL
```

Example Output

```text
/home/natraj
natraj
/home/natraj
/usr/local/bin:/usr/bin:/bin
/bin/bash
```

---

# Read-only Variables

A **read-only variable** cannot be modified or deleted after it is created.

Syntax

```bash
readonly variable=value
```

Example

```bash
readonly company="Infosys"

echo $company
```

Attempt to modify

```bash
company="TCS"
```

Output

```text
bash: company: readonly variable
```

Check read-only variables

```bash
readonly
```

---

# Export Variables

Normally, variables are available only in the current shell.

Using `export` makes a variable available to child processes.

Without export

```bash
name=Natraj
bash

echo $name
```

Output

```text

```

(No output)

---

With export

```bash
export name=Natraj

bash

echo $name
```

Output

```text
Natraj
```

View exported variables

```bash
export
```

---

# Local Variable vs Export Variable

| Local Variable     | Export Variable                 |
| ------------------ | ------------------------------- |
| Current shell only | Current shell + child processes |
| Created normally   | Created using `export`          |
| Not inherited      | Inherited by child shells       |

Example

```bash
city=Pune
export country=India
```

# Special Variables

Special variables provide information about the current script, arguments, process IDs, and execution status.

---

## `$0`

Displays the script name.

Example

```bash
#!/bin/bash

echo $0
```

Run

```bash
bash demo.sh
```

Output

```text
demo.sh
```

---

## `$1 - $9`

Represents positional command-line arguments.

Example

```bash
#!/bin/bash

echo "First Argument : $1"
echo "Second Argument : $2"
echo "Third Argument : $3"
```

Run

```bash
bash demo.sh Linux DevOps AWS
```

Output

```text
First Argument : Linux
Second Argument : DevOps
Third Argument : AWS
```

---

## `$#`

Displays the total number of command-line arguments.

Example

```bash
#!/bin/bash

echo $#
```

Run

```bash
bash demo.sh one two three
```

Output

```text
3
```

---

## `$*`

Displays all arguments as a **single string**.

Example

```bash
echo $*
```

Run

```bash
bash demo.sh Linux DevOps AWS
```

Output

```text
Linux DevOps AWS
```

---

## `$@`

Displays all arguments individually.

Example

```bash
echo "$@"
```

Run

```bash
bash demo.sh Linux DevOps AWS
```

Output

```text
Linux DevOps AWS
```

### Difference (`"$*"` vs `"$@"`)

```bash
for i in "$*"
do
    echo "$i"
done
```

Output

```text
Linux DevOps AWS
```

(One single item)

---

```bash
for i in "$@"
do
    echo "$i"
done
```

Output

```text
Linux
DevOps
AWS
```

(Each argument is treated separately)

---

## `$$`

Displays the Process ID (PID) of the current shell/script.

Example

```bash
echo $$
```

Output

```text
24561
```

Check with

```bash
ps -p $$
```

---

## `$?`

Displays the exit status of the previously executed command.

Example

```bash
pwd

echo $?
```

Output

```text
0
```

Failure Example

```bash
cd /abc

echo $?
```

Output

```text
1
```

---

## `$!`

Displays the Process ID (PID) of the **last background process**.

Example

```bash
sleep 100 &
```

```bash
echo $!
```

Output

```text
31580
```

Verify

```bash
ps -p $!
```

# Complete Example

```bash
#!/bin/bash

echo "Script Name      : $0"
echo "First Argument   : $1"
echo "Second Argument  : $2"
echo "Total Arguments  : $#"
echo "All Arguments(*) : $*"
echo "All Arguments(@) : $@"
echo "Current PID      : $$"

pwd

echo "Exit Status      : $?"
```

Run

```bash
bash variables.sh Linux DevOps
```

Output

```text
Script Name      : variables.sh
First Argument   : Linux
Second Argument  : DevOps
Total Arguments  : 2
All Arguments(*) : Linux DevOps
All Arguments(@) : Linux DevOps
Current PID      : 26485
/home/natraj
Exit Status      : 0
```

# Best Practices

- Use meaningful variable names (`backup_dir`, `server_name`, `log_file`).
- Use quotes around variables to handle spaces:

```bash
echo "$name"
```

- Use `readonly` for constants that should not change.
- Use `export` only when child processes need the variable.
- Always check `$?` after critical commands.

# Interview Questions

### 1. What is a variable in Shell Scripting?

A variable stores data that can be reused in a script.

### 2. What is the difference between a user variable and an environment variable?

- **User Variable:** Created by the user and available only in the current shell unless exported.
- **Environment Variable:** Predefined or exported variables inherited by child processes.

### 3. What is the purpose of `export`?

It makes a variable available to child shells and processes.

### 4. What is a read-only variable?

A variable that cannot be modified or unset after creation using `readonly`.

### 5. What is the difference between `$*` and `$@`?

- `"$*"` treats all arguments as **one string**.
- `"$@"` treats each argument as a **separate string**, making it safer in loops.

### 6. What does `$$` return?

The Process ID (PID) of the current shell or script.

### 7. What does `$!` return?

The PID of the most recently started background process.

### 8. What does `$?` return?

The exit status of the previously executed command (`0` = success, non-zero = error).
