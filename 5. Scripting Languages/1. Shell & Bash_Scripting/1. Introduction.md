## Linux Architecture

```text
+---------------------------+
|        User               |
+---------------------------+
            │
            ▼
+---------------------------+
|    Applications           |
| (vim, git, docker, etc.)  |
+---------------------------+
            │
            ▼
+---------------------------+
|         Shell             |
| (bash, sh, zsh, ksh)      |
+---------------------------+
            │
            ▼
+---------------------------+
|        Kernel             |
+---------------------------+
            │
            ▼
+---------------------------+
|       Hardware            |
| CPU, RAM, Disk, Network   |
+---------------------------+
```

### Explanation

- **User** → Gives commands.
- **Applications** → Programs like Git, Docker, Vim, etc.
- **Shell** → Takes user commands and sends them to the Kernel.
- **Kernel** → Core of Linux; manages CPU, memory, files, devices, and processes.
- **Hardware** → Physical components such as CPU, RAM, storage, and network.

---

# What is Shell?

A **Shell** is a **Command Line Interpreter (CLI)** that acts as an interface between the **User** and the **Linux Kernel**.

It receives commands from the user, passes them to the kernel, and displays the output.

### Example

```bash
pwd
```

Output

```text
/home/natraj
```

# What is Bash?

**Bash (Bourne Again Shell)** is the most popular Linux shell.

It is the default shell on most Linux distributions.

Bash is used for:

- Running Linux commands
- Writing Shell Scripts
- Automating tasks
- System administration
- DevOps automation

Default path

```bash
/bin/bash
```

Check current shell

```bash
echo $SHELL
```

Example Output

```text
/bin/bash
```

# History of Shell

### 1. Bourne Shell (sh)

- Developed by **Stephen Bourne**
- Introduced in **1979**
- Default executable

```text
/bin/sh
```

Limitations

- Fewer scripting features
- No command history
- No command auto-completion

---

### 2. Bourne Again Shell (bash)

- Released in **1989**
- Created as an improved version of Bourne Shell
- GNU Project
- Default shell in most Linux distributions

Default executable

```text
/bin/bash
```

Improvements

- Command History
- Auto-completion (Tab)
- Aliases
- Arrays
- Better scripting
- Functions
- Command Editing
- Better Variables

---

# Types of Shells

| Shell | Full Name                  | Default Path  | Features               |
| ----- | -------------------------- | ------------- | ---------------------- |
| sh    | Bourne Shell               | /bin/sh       | Original UNIX Shell    |
| bash  | Bourne Again Shell         | /bin/bash     | Most Popular           |
| ksh   | Korn Shell                 | /bin/ksh      | Fast scripting         |
| csh   | C Shell                    | /bin/csh      | C-like syntax          |
| tcsh  | TENEX C Shell              | /bin/tcsh     | Improved csh           |
| zsh   | Z Shell                    | /bin/zsh      | Powerful customization |
| fish  | Friendly Interactive Shell | /usr/bin/fish | Beginner friendly      |

---

# Shell vs Bash

| Shell                       | Bash                     |
| --------------------------- | ------------------------ |
| Generic command interpreter | One type of Shell        |
| Many shell types available  | Most commonly used shell |
| Basic scripting             | Advanced scripting       |
| Less powerful               | More powerful            |
| Example: sh, ksh, zsh       | bash only                |

## Key Interview Points

- **Shell** is an interface between the **User** and the **Kernel**.
- **Kernel** is the core of the Linux operating system.
- **Bash** stands for **Bourne Again Shell**.
- **Stephen Bourne** developed the original **Bourne Shell (sh)**.
- **Bash** was developed by the **GNU Project** as an enhanced replacement for **sh**.
- The default Bash executable is **`/bin/bash`**, while the Bourne Shell is **`/bin/sh`**.
- **`history`** displays previously executed commands.
- Use **`echo $SHELL`** to check your current login shell.
- Use **`cat /etc/shells`** to list all installed login shells.
