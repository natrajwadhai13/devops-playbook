# What is a Shell Script?

A **Shell Script** is a file that contains one or more Linux commands.

Instead of typing commands one by one, you can save them in a file and execute them together.

- File extension is usually **`.sh`** (optional but recommended).
- Executed by a shell such as **Bash**.

---

# First Shell Script

### Step 1: Create a Script

```bash
vim first_script.sh
```

or

```bash
touch first_script.sh
```

---

### Step 2: Write the Script

```bash
#!/bin/bash

echo "Hello World"
echo "Welcome to Shell Scripting"
date
pwd
```

---

### Step 3: Save the File

For **vim**:

```text
Esc
:wq
```

---

### Step 4: Execute the Script

```bash
bash first_script.sh
```

Output

```text
Hello World
Welcome to Shell Scripting
Sat Jul 19 20:30:10 IST 2026
/home/natraj
```

# Shebang (`#!/bin/bash`)

The **Shebang** tells Linux which interpreter should execute the script.

Syntax

```bash
#!/bin/bash
```

Meaning

- `#!` → Shebang
- `/bin/bash` → Bash interpreter

Example

```bash
#!/bin/bash

echo "Running with Bash"
```

Without a shebang, the current shell may execute the script, which can lead to unexpected behavior if the script uses Bash-specific features.

---

# Comments

Comments are ignored during execution and are used to explain the code.

## Single Line Comment

```bash
# This is a comment

echo "Hello"
```

---

## Multiple Single-Line Comments

```bash
# Author : Natraj
# Version : 1.0
# Date : 19-Jul-2026

echo "DevOps"
```

> **Note:** Bash does **not** have a true multi-line comment syntax. Many people use multiple `#` lines. Another common trick is shown below.

### Multi-line Comment Trick

```bash
: '
This is a
multi-line comment.
It is commonly used in Bash,
but it is actually a no-op command.
'
```

# Execute a Script

There are multiple ways to execute a shell script.

## Method 1

```bash
bash first_script.sh
```

Uses the Bash interpreter directly.

---

## Method 2

```bash
sh first_script.sh
```

Uses the `sh` interpreter.

> Avoid this if your script contains Bash-specific features (arrays, `[[ ]]`, etc.).

---

## Method 3

```bash
./first_script.sh
```

Runs the script directly.

Requires:

- Execute permission
- Correct shebang (`#!/bin/bash`)

---

# chmod +x

By default, a script is **not executable**.

Grant execute permission:

```bash
chmod +x first_script.sh
```

Check permissions:

```bash
ls -l first_script.sh
```

Example Output

```text
-rwxr-xr-x 1 root root 80 Jul 19 20:40 first_script.sh
```

Here:

- `r` = Read
- `w` = Write
- `x` = Execute

---

# `bash script.sh` vs `./script.sh`

| `bash script.sh`                                               | `./script.sh`                                                  |
| -------------------------------------------------------------- | -------------------------------------------------------------- |
| Executes the script using the Bash interpreter                 | Executes the script directly                                   |
| Does **not** require execute permission                        | Requires execute permission (`chmod +x`)                       |
| Shebang is not required (because Bash is explicitly specified) | Shebang is recommended to tell the OS which interpreter to use |
| Useful for testing                                             | Common for production scripts                                  |

### Example

Without execute permission:

```bash
bash first_script.sh
```

Works ✅

But

```bash
./first_script.sh
```

Output

```text
Permission denied
```

After:

```bash
chmod +x first_script.sh
```

Now:

```bash
./first_script.sh
```

Works successfully.

# Exit Status (`$?`)

Every Linux command returns an **Exit Status**.

Check it using:

```bash
echo $?
```

---

## Success

```bash
pwd
echo $?
```

Output

```text
0
```

`0` means the command executed successfully.

---

## Failure

```bash
cd /abc
echo $?
```

Output

```text
bash: cd: /abc: No such file or directory

1
```

A non-zero value indicates an error.

---

## Common Exit Codes

| Exit Code | Meaning                                       |
| --------- | --------------------------------------------- |
| `0`       | Success                                       |
| `1`       | General Error                                 |
| `2`       | Incorrect Usage / Misuse of Built-in Commands |
| `126`     | Permission denied (cannot execute)            |
| `127`     | Command not found                             |
| `130`     | Script terminated using `Ctrl + C`            |

# Creating a Script (Complete Example)

```bash
#!/bin/bash

# Simple Demo Script

echo "User : $(whoami)"
echo "Date : $(date)"
echo "Current Directory : $(pwd)"

ls

echo "Script Completed Successfully"
```

Run:

```bash
chmod +x demo.sh
./demo.sh
```

---

# Best Practices

- Always start scripts with `#!/bin/bash`.
- Give scripts meaningful names (e.g., `backup.sh`, `health_check.sh`).
- Add comments to explain important logic.
- Check exit status after critical commands.
- Use `chmod +x` only when you want the script to be directly executable.
- Test scripts with `bash script.sh` before making them executable.

# Interview Questions

### 1. What is a shell script?

A file containing Linux commands executed by a shell interpreter.

### 2. What is the purpose of `#!/bin/bash`?

It specifies that the script should be executed using the Bash interpreter.

### 3. What is the difference between `bash script.sh` and `./script.sh`?

- `bash script.sh` explicitly uses Bash and does not require execute permission.
- `./script.sh` runs the script directly, requires execute permission, and uses the shebang to determine the interpreter.

### 4. What does `chmod +x` do?

It adds execute permission to a file.

### 5. What does `$?` represent?

It stores the exit status of the previously executed command.

### 6. What does exit code `0` mean?

The command completed successfully.

### 7. What does exit code `127` mean?

The specified command was not found.
