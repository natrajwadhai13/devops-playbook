## рЯІ≠ **YAML Roadmap for DevOps Engineers (2вАУ3 Weeks)**

рЯУШ **Goal**: Be confident writing and reading YAML used in real-world DevOps tools like Ansible, Kubernetes, GitHub Actions, GitLab CI, Docker Compose, etc.

---

## рЯФ∞ **Week 1: YAML Fundamentals**

### рЯОѓ Outcomes:

- Learn YAML syntax, structure, and common mistakes

### Topics:

- What is YAML?
- YAML vs JSON
- Syntax: indentation, key-value pairs, lists, dictionaries
- Scalars, strings, booleans, nulls
- Anchors (`&`) and aliases (`*`)
- Comments (`#`), multi-line strings (`|`, `>`)

### Hands-on:

- Convert JSON to YAML (and vice versa)
- Write simple YAML examples:

  ```yaml
  name: natraj
  skills:
    - DevOps
    - Linux
    - Cloud
  location:
    city: Pune
    area: Kharadi
  ```

## вЪЩпЄП **Week 2: Tool-Specific YAML Use**

### рЯОѓ Outcomes:

- Understand how YAML is used in popular DevOps tools

---

### рЯФє **Ansible YAML (Playbooks & Roles)**

```yaml
- name: Install Nginx
  hosts: web
  become: yes
  tasks:
    - name: Install package
      apt:
        name: nginx
        state: present
```

рЯУМ Learn:

- `hosts`, `tasks`, `vars`, `roles`, `handlers`
- How indentation impacts execution

---

### рЯФє **Kubernetes YAML (Manifests)**

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
    - name: nginx
      image: nginx
```

рЯУМ Learn:

- `apiVersion`, `kind`, `metadata`, `spec`
- Deployment, Service, Ingress YAML files

---

### рЯФє **Docker Compose YAML**

```yaml
version: "3"
services:
  web:
    image: nginx
    ports:
      - "80:80"
```

рЯУМ Learn:

- `version`, `services`, `volumes`, `networks`

---

### рЯФє **GitLab CI/CD YAML**

```yaml
stages:
  - build
  - deploy

build:
  stage: build
  script:
    - echo "Building"

deploy:
  stage: deploy
  script:
    - echo "Deploying"
```

рЯУМ Learn:

- `stages`, `script`, `before_script`, `only`, `except`, `rules`

## вЬЕ Best Practices

- Always use **2 spaces**, never tabs
- Use **inline comments** to describe blocks
- Validate YAML using tools like:
  - [https://www.yamllint.com](https://www.yamllint.com)
  - `yamllint` CLI
  - VSCode extension: YAML by Red Hat

## рЯУШ YAML Cheat Sheet Summary

| Feature      | Example                                   |                                   |
| ------------ | ----------------------------------------- | --------------------------------- |
| Key-Value    | `name: natraj`                            |                                   |
| List         | `skills: [aws, docker, git]`              |                                   |
| Nested List  | `skills: \n  - aws \n  - docker`          |                                   |
| Boolean      | `enabled: true`                           |                                   |
| Multiline    | \`desc:                                   | \n This is a long \n text block\` |
| Anchor/Alias | `default: &def \n  val: 1 \n reuse: *def` |                                   |
