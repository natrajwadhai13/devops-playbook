# YAML (YAML Ain't Markup Language)

## What is YAML?

YAML (**YAML Ain't Markup Language**) is a **human-readable data serialization language** used to store configuration and exchange data between applications.

It is widely used because it is easy to read and write.

### Common Uses

- Kubernetes manifests
- Docker Compose
- Ansible Playbooks
- GitHub Actions
- CI/CD Pipelines (GitLab, Jenkins)
- Cloud Configuration
- Application Configuration Files

---

# Features

- Easy to read
- Uses indentation instead of brackets
- Supports comments
- Stores structured data
- Supports lists, objects, strings, numbers, booleans, null values, timestamps, etc.

---

# File Extension

```text
.yaml
.yml
```

Example

```text
config.yaml
docker-compose.yml
deployment.yaml
```

---

# YAML Document Start

A YAML file can start with:

```yaml
---
```

This indicates the **beginning of a YAML document**.

Example

```yaml
---
name: Natraj
age: 25
```

---

# Multiple Documents

A single YAML file can contain multiple documents.

Example

```yaml
---
name: Dev
---
name: Prod
---
name: Test
```

Each document is separated by

```yaml
---
```

---

# Comments

Comments are ignored by YAML.

Use

```yaml
#
```

Example

```yaml
# Database Configuration
host: localhost
port: 3306
```

> **Note:** `//` is **not an official YAML comment syntax**. Only `#` is valid in standard YAML.

---

# Key-Value Pair

Basic syntax

```yaml
name: Natraj

```

Output

```text
name → Natraj

```

# Objects (Dictionary)

Objects contain multiple key-value pairs.

```yaml
server:
  host: localhost
  port: 8080
  storage: 50GB
```

Equivalent JSON

```json
{
  "server": {
    "host": "localhost",
    "port": 8080,
    "storage": "50GB"
  }
}
```

---

# Nested Objects

Objects can contain other objects.

```yaml
database:
  mysql:
    host: localhost
    port: 3306
```

# Lists

Lists are created using `-`.

```yaml

  - Python
  - Java
  - Go
```


# Multiline String (Literal)

Use `|` when you want to preserve line breaks exactly.

```yaml
brewing_instruction: |
  Boil water.
  Add tea leaves.
  Add milk.
  Serve hot.
```

Output

```text
Boil water.
Add tea leaves.
Add milk.
Serve hot.
```

---

# Folded String

Use `>` to convert multiple lines into one line.

```yaml
description: >
  YAML is simple
  and easy
  to learn.
```

Output

```text
YAML is simple and easy to learn.
```

String

```yaml

```

Integer


# Anchors (&) एंकर

Anchors let you define data once and reuse it.

- `&` creates an anchor.
- `*` reuses the anchor.
- `<<` merges the anchored object into another object.

```yaml
default: &server
  host: localhost
  port: 8080
  timeout: 60

development:
  <<: *server

production:
  <<: *server
```

Result

```text
development
  host: localhost
  port: 8080
  timeout: 60

production
  host: localhost
  port: 8080
  timeout: 60
```

---

# Aliases (\*)

Reuse previously defined values.

```yaml
database: &db
  host: localhost
  port: 3306

backup:
  <<: *db
```

---

# Merge Key (`<<`)

Merge one object into another.

```yaml
defaults: &default
  retries: 3
  timeout: 60

service:
  <<: *default
  name: API
```

Output

```yaml
service:
  retries: 3
  timeout: 60
  name: API
```

---

# Sets

Stores unique values.

```yaml
unique_spices: !!set
  ? ginger
  ? cloves
  ? cardamom
```

Duplicate values are ignored.

---

# Indentation Rules

YAML uses **spaces**, not brackets.

✅ Correct

```yaml
server:
  host: localhost
  port: 8080
```

❌ Wrong

```yaml
server:
 host: localhost
    port: 8080
```

### Best Practices

- Use **2 spaces** (most common)
- Do **not use Tabs**
- Keep indentation consistent
- Child elements must be indented more than their parent

# Reading YAML in Python

Install package


```python

    data = yaml.safe_load(file)
```

# YAML Interview Questions

### 1. What is YAML?

A human-readable data serialization language mainly used for configuration files.

### 2. What is the difference between YAML and JSON?

YAML is easier to read, supports comments and anchors, while JSON uses braces and does not support comments.

### 3. What is the purpose of `---`?

Marks the beginning of a YAML document and separates multiple documents in one file.

### 4. What is the difference between `|` and `>`?

- `|` preserves line breaks.
- `>` folds multiple lines into a single line.

### 5. What are Anchors and Aliases?

- `&` creates an anchor.
- `*` reuses the anchor.
- `<<` merges the anchored object into another object.

### 6. How are lists created?

Using `-`

```yaml
skills:
  - Linux
  - Docker
```

### 7. How do you represent null?

```yaml
value: null
```

or

```yaml
value: ~
```

### 8. Can YAML contain multiple documents?

Yes.

```yaml
---
Document 1
---
Document 2
```

### 9. Can YAML contain comments?

Yes.

```yaml
# This is a comment
```

### 10. Why is indentation important in YAML?

Indentation defines the hierarchy of data. Incorrect indentation can make the YAML invalid.
