# ЁЯФР **Azure IAM (Identity & Access Management)**

---

## 1я╕ПтГг **What is Azure IAM?**

**Azure IAM** controls **access to Azure resources** by managing:

* **Who** can sign in (Authentication)
* **What** they can do (Authorization)

Azure IAM is built using **Microsoft Entra ID**
(formerly **Azure Active Directory / AAD**).

---

## 2я╕ПтГг **Core IAM Concepts (Foundation)**

### ЁЯФС Authentication vs ЁЯЫВ Authorization

| Concept            | Meaning                                 |
| ------------------ | --------------------------------------- |
| **Authentication** | Verifies identity (*Who are you?*)      |
| **Authorization**  | Grants permissions (*What can you do?*) |

ЁЯУМ **Exam Trap:**
Authentication тЙа Authorization

---

## 3я╕ПтГг **Authentication (Users, Groups, Sign-in)**

### ЁЯСд **Users**

* Individual human identities
* Used to login to:

  * Azure Portal
  * Azure CLI (`az login`)
* Authentication methods:

  * Username + password
  * **MFA (Multi-Factor Authentication)**

**Practical (Portal):**

```
Azure Portal тЖТ Microsoft Entra ID тЖТ Users тЖТ New User
```

---

### ЁЯСе **Groups**

* Collection of users
* Used for **easy permission management**

**Why Groups?**

* Assign role once to group
* All users inherit access

**Types**

* Security Groups (most common)
* Microsoft 365 Groups

**Best Practice**

> Assign roles to **groups**, not individual users.

**Practical:**

```
Entra ID тЖТ Groups тЖТ New Group тЖТ Add Members
```

---

## 4я╕ПтГг **Microsoft Entra ID (Azure AD / AAD)**

### ЁЯФР What is Entra ID?

* Cloud-based **identity & access service**
* Manages:

  * Users
  * Groups
  * Applications
  * Authentication

### Used for:

* Azure Portal login
* Azure CLI login
* Single Sign-On (SSO)
* Application authentication

ЁЯУМ **Important**

* Entra ID тЙа On-prem Active Directory
* Entra ID is **cloud-native**

---

## 5я╕ПтГг **Authorization (RBAC тАУ Role-Based Access Control)**

### ЁЯз╛ What is RBAC?

RBAC controls **what actions** an identity can perform on Azure resources.

---

### ЁЯОн RBAC Components (Very Important)

1. **Security Principal** тАУ Who (User / Group / Service Principal / Managed Identity)
2. **Role Definition** тАУ What actions are allowed
3. **Scope** тАУ Where access applies

---

### ЁЯП╖я╕П **Built-in Roles (AZ-900 MUST-KNOW)**

| Role            | Permissions                       |
| --------------- | --------------------------------- |
| **Owner**       | Full access + assign roles        |
| **Contributor** | Full access (cannot assign roles) |
| **Reader**      | View-only                         |

ЁЯУМ **Exam Line**

> Only **Owner** can assign RBAC roles.

---

### ЁЯУН **RBAC Scope Levels**

* Management Group
* Subscription
* Resource Group
* Resource

ЁЯУМ Permissions flow **top тЖТ down**.

**Practical:**

```
Resource Group тЖТ Access Control (IAM) тЖТ Add role assignment
```

---

## 6я╕ПтГг **Service Principal (Application Identity)**

### ЁЯдЦ What is a Service Principal?

* Identity for **applications or automation**
* Used by:

  * CI/CD pipelines
  * Automation scripts
  * External apps

### Why needed?

* Applications cannot login like users
* Secure, controlled access using RBAC

**Example**

> Azure DevOps pipeline deploying resources using a Service Principal.

**Practical (CLI):**

```bash
az ad sp create-for-rbac --name myApp
```

---

## 7я╕ПтГг **Managed Identities (Most Secure тАУ Azure Recommended)**

### ЁЯФД What is a Managed Identity?

* Azure-managed identity for **Azure resources**
* No passwords, no secrets
* Automatically handled by Azure

---

### ЁЯзй Types of Managed Identity

| Type                | Description          |
| ------------------- | -------------------- |
| **System-assigned** | Tied to one resource |
| **User-assigned**   | Standalone, reusable |

---

### тЬЕ Why Managed Identity?

* No credential management
* Highly secure
* Best for Azure-to-Azure access

**Example Use Cases**

* VM accessing Azure Storage
* App Service accessing Key Vault
* Function App accessing Database

**Practical Flow**

```
Enable Managed Identity тЖТ Assign RBAC Role тЖТ Access Resource
```

ЁЯУМ **Exam Line**

> Managed Identity is the **best way** for Azure services to access other Azure services.

---

## 8я╕ПтГг **Security Features in Azure IAM**

### ЁЯФР Key Security Controls

* **MFA** тАУ Extra login verification
* **Conditional Access** тАУ Rules based on:

  * Location
  * Device
  * Risk
* **Least Privilege** тАУ Minimum access
* **RBAC** тАУ Controlled authorization
* **Key Vault** тАУ Secure secrets

---
