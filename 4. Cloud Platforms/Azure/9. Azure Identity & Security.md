# üîê **Azure IAM (Identity & Access Management)**

---

## 1Ô∏è‚É£ **What is Azure IAM?**

**Azure IAM** controls **access to Azure resources** by managing:

* **Who** can sign in (Authentication)
* **What** they can do (Authorization)

Azure IAM is built using **Microsoft Entra ID**
(formerly **Azure Active Directory / AAD**).

---

## 2Ô∏è‚É£ **Core IAM Concepts (Foundation)**

### üîë Authentication vs üõÇ Authorization

| Concept            | Meaning                                 |
| ------------------ | --------------------------------------- |
| **Authentication** | Verifies identity (*Who are you?*)      |
| **Authorization**  | Grants permissions (*What can you do?*) |

üìå **Exam Trap:**
Authentication ‚â† Authorization

---

## 3Ô∏è‚É£ **Authentication (Users, Groups, Sign-in)**

### üë§ **Users**

* Individual human identities
* Used to login to:

  * Azure Portal
  * Azure CLI (`az login`)
* Authentication methods:

  * Username + password
  * **MFA (Multi-Factor Authentication)**

**Practical (Portal):**

```
Azure Portal ‚Üí Microsoft Entra ID ‚Üí Users ‚Üí New User
```

---

### üë• **Groups**

* Collection of users
* Used for **easy permission management**

**Why Groups?**

* Assign role once to group
* All users inherit access

**Types**

* Security Groups (most common)
* Microsoft 365 Groups

**Best Practice**

> Assign roles to **groups**, not individual users.

**Practical:**

```
Entra ID ‚Üí Groups ‚Üí New Group ‚Üí Add Members
```

---

## 4Ô∏è‚É£ **Microsoft Entra ID (Azure AD / AAD)**

### üîê What is Entra ID?

* Cloud-based **identity & access service**
* Manages:

  * Users
  * Groups
  * Applications
  * Authentication

### Used for:

* Azure Portal login
* Azure CLI login
* Single Sign-On (SSO)
* Application authentication

üìå **Important**

* Entra ID ‚â† On-prem Active Directory
* Entra ID is **cloud-native**

---

## 5Ô∏è‚É£ **Authorization (RBAC ‚Äì Role-Based Access Control)**

### üßæ What is RBAC?

RBAC controls **what actions** an identity can perform on Azure resources.

---

### üé≠ RBAC Components (Very Important)

1. **Security Principal** ‚Äì Who (User / Group / Service Principal / Managed Identity)
2. **Role Definition** ‚Äì What actions are allowed
3. **Scope** ‚Äì Where access applies

---

### üè∑Ô∏è **Built-in Roles (AZ-900 MUST-KNOW)**

| Role            | Permissions                       |
| --------------- | --------------------------------- |
| **Owner**       | Full access + assign roles        |
| **Contributor** | Full access (cannot assign roles) |
| **Reader**      | View-only                         |

üìå **Exam Line**

> Only **Owner** can assign RBAC roles.

---

### üìç **RBAC Scope Levels**

* Management Group
* Subscription
* Resource Group
* Resource

üìå Permissions flow **top ‚Üí down**.

**Practical:**

```
Resource Group ‚Üí Access Control (IAM) ‚Üí Add role assignment
```

---

## 6Ô∏è‚É£ **Service Principal (Application Identity)**

### ü§ñ What is a Service Principal?

* Identity for **applications or automation**
* Used by:

  * CI/CD pipelines
  * Automation scripts
  * External apps

### Why needed?

* Applications cannot login like users
* Secure, controlled access using RBAC

**Example**

> Azure DevOps pipeline deploying resources using a Service Principal.

**Practical (CLI):**

```bash
az ad sp create-for-rbac --name myApp
```

---

## 7Ô∏è‚É£ **Managed Identities (Most Secure ‚Äì Azure Recommended)**

### üîÑ What is a Managed Identity?

* Azure-managed identity for **Azure resources**
* No passwords, no secrets
* Automatically handled by Azure

---

### üß© Types of Managed Identity

| Type                | Description          |
| ------------------- | -------------------- |
| **System-assigned** | Tied to one resource |
| **User-assigned**   | Standalone, reusable |

---

### ‚úÖ Why Managed Identity?

* No credential management
* Highly secure
* Best for Azure-to-Azure access

**Example Use Cases**

* VM accessing Azure Storage
* App Service accessing Key Vault
* Function App accessing Database

**Practical Flow**

```
Enable Managed Identity ‚Üí Assign RBAC Role ‚Üí Access Resource
```

üìå **Exam Line**

> Managed Identity is the **best way** for Azure services to access other Azure services.

---

## 8Ô∏è‚É£ **Security Features in Azure IAM**

### üîê Key Security Controls

* **Least Privilege** ‚Äì Minimum access
* **RBAC** ‚Äì Controlled authorization
* **Key Vault** ‚Äì Secure secrets
