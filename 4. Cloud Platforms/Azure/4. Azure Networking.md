## **1Ô∏è‚É£ Virtual Network (VNet)**

* Private network in Azure (similar to on-prem network)
* Uses **CIDR IP range** (ex: 10.0.0.0/16)

* Logical division of a VNet

* Subnet must be inside VNet address space

* Applied to:

  * Subnet

# **Azure DDoS Network Protection & DNAT**

---

## **üîπ Azure DDoS Network Protection**

* Protects Azure resources from **Distributed Denial of Service (DDoS)** attacks.
* Monitors abnormal traffic and automatically mitigates attacks.
* Works at **network layer (L3/L4)**.

### **Types**

* **Basic** ‚Üí Free, enabled by default
* **Standard** ‚Üí Paid, advanced protection

### **DDoS Standard Features**

* Real-time attack detection
* Automatic mitigation
* Traffic analytics & alerts
* Cost protection during attack
* Protects public IP resources (VMs, LB, App Gateway)

### **Key Points**

* Enabled at **VNet level**
* Protects all public IPs in that VNet
* No application changes required

---

## **üîπ DNAT (Destination Network Address Translation)**

* Translates **public IP + port ‚Üí private IP + port**
* Used to allow inbound internet traffic to private resources.

### **Where DNAT is Used**

* Azure Firewall
* Azure Load Balancer
* Network Virtual Appliances (NVA)

### **Example**

```
Public IP: 20.10.10.5:80
        ‚Üì DNAT
Private IP: 10.0.1.4:80
```

### **Use Cases**

* Exposing VM to internet
* Publishing internal services
* Port forwarding

---

## **üß† Interview One-Liners**

* **DDoS** protects against traffic flooding attacks
* **DNAT** forwards external traffic to internal resources
* DDoS ‚â† Firewall
* DNAT works only for **inbound traffic**
