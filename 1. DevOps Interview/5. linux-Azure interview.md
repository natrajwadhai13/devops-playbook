## üîπ SYSTEM & ARCHITECTURE

### 1. Explain the Linux boot process in detail.

**Answer:**

1. **BIOS/UEFI** ‚Äì Hardware initialization
2. **MBR / GPT** ‚Äì Finds bootloader
3. **GRUB** ‚Äì Loads kernel and initramfs
4. **Kernel** ‚Äì Initializes hardware, mounts root filesystem
5. **init / systemd** ‚Äì Starts system services

---

### 2. What is `systemd` and why is it used?

**Answer:**
`systemd` is the **init system** that manages services, targets, logging, and dependencies.
It is faster and dependency-aware compared to SysVinit.

---

### 3. Difference between Runlevels and Targets?

**Answer:**
Runlevels are legacy (SysV), targets are used by **systemd**.

Example:

```bash
multi-user.target ‚Üí runlevel 3
graphical.target ‚Üí runlevel 5
```

---

## üîπ PERFORMANCE & TROUBLESHOOTING

### 4. Production server is slow. What will you check first?

**Answer:**

1. CPU ‚Üí `top`, `htop`
2. Memory ‚Üí `free -m`
3. Disk ‚Üí `df -h`, `iostat`
4. Processes ‚Üí `ps aux`
5. Logs ‚Üí `/var/log/messages`, `journalctl`

---

### 5. How do you find which process is consuming high memory?

**Answer:**

```bash
ps aux --sort=-%mem | head
```

---

### 6. What is load average?

**Answer:**
Load average shows the **number of processes waiting for CPU** in:

- 1 minute
- 5 minutes
- 15 minutes

Rule:

- Load ‚â§ CPU cores ‚Üí OK
- Load > CPU cores ‚Üí Performance issue

---

### 7. Difference between CPU idle and CPU wait?

**Answer:**

- **Idle:** CPU has nothing to do
- **I/O wait:** CPU waiting for disk/network operations

---

## üîπ PROCESS & MEMORY MANAGEMENT

### 8. What is a zombie process?

**Answer:**
A zombie process has completed execution but still exists because the parent hasn‚Äôt read its exit status.

---

### 9. How do you kill a stuck process?

**Answer:**

```bash
kill -9 PID
```

---

### 10. What is OOM Killer?

**Answer:**
When memory is exhausted, Linux automatically kills processes using **OOM Killer** to keep the system alive.

---

## üîπ FILESYSTEM & STORAGE

### 11. Difference between ext4 and xfs?

**Answer:**

| ext4                 | xfs                  |
| -------------------- | -------------------- |
| General purpose      | High performance     |
| Good for small files | Best for large files |
| Slower resize        | Fast scaling         |

---

### 12. How do you extend a filesystem without downtime?

**Answer:**
For LVM:

```bash
lvextend -L +10G /dev/vg/lv
resize2fs /dev/vg/lv
```

---

### 13. What is inode?

**Answer:**
Inode stores **metadata** (permissions, owner, size, pointers).
Filename is NOT stored in inode.

---

### 14. Disk is free but system says ‚ÄúNo space left‚Äù. Why?

**Answer:**

- Inode exhaustion
- Hidden deleted files still open by processes
  Check:

```bash
df -i
lsof | grep deleted
```

5. Firewall ‚Äì `iptables`, `firewalld`

- SELinux/AppArmor

### 19. Difference between SELinux and AppArmor?

**Answer:**

### 20. How do you find who logged into the server?

```

## üîπ LOGGING & MONITORING

### 21. Where are system logs stored?

**Answer:**

- `/var/log/messages`
- `/var/log/syslog`

---

### 22. How do you monitor logs in real time?

**Answer:**

```

---

## üîπ AUTOMATION & SCRIPTING

### 23. Difference between `cron` and `systemd timers`?

**Answer:**

- Cron ‚Üí Time-based
- systemd timer ‚Üí Dependency-based, more reliable

---

### 24. Write a script to check disk usage and alert if >80%.

**Answer:**

```bash
#!/bin/bash
usage=$(df / | awk 'NR==2 {print $5}' | cut -d% -f1)
if [ $usage -gt 80 ]; then
  echo "Disk usage critical"
fi
```

## üîπ REAL PRODUCTION SCENARIOS

### 25. Server rebooted automatically. How will you investigate?

**Answer:**

- `last reboot`
- Check logs ‚Üí `journalctl -b -1`
- Check OOM ‚Üí `/var/log/messages`
- Hardware alerts / cloud events

---

### 26. Application works manually but fails via cron. Why?

**Answer:**

- Cron has limited PATH
- Environment variables missing
- Permission issues

---

### 27. How do you debug a service not starting?

**Answer:**

```bash
systemctl status service
journalctl -u service
```

## üîπ Basic Ansible Interview Questions

### 1. What is Ansible?

**Answer:**
Ansible is an **open-source automation tool** used for configuration management, application deployment, and orchestration.

---

### 2. Why is Ansible agentless?

**Answer:**
Ansible uses **SSH (for Linux)** and **WinRM (for Windows)** to communicate with servers, so no agent installation is required on managed nodes.

---

### 3. What are the main components of Ansible?

**Answer:**

- Inventory
- Playbook
- Modules
- Tasks
- Roles
- Handlers
- Facts

---

### 4. What is an Inventory file?

**Answer:**
An inventory file contains the **list of target servers** (IP addresses or hostnames) managed by Ansible.

Example:

```ini
[web]
192.168.1.10
192.168.1.11
```

---

### 5. What is a Playbook?

**Answer:**
A playbook is a **YAML file** that defines a set of tasks to be executed on managed hosts.

---

### 6. What is YAML?

**Answer:**
YAML stands for **Yet Another Markup Language**.
It is a human-readable data serialization format based on indentation.

---

## üîπ Intermediate Ansible Interview Questions

### 7. What is the difference between a Module and a Task?

**Answer:**

- **Module:** A reusable unit that performs a specific action (copy, yum, service).
- **Task:** Executes a module with specific parameters.

---

### 8. What are Handlers?

**Answer:**
Handlers are special tasks that run **only when notified**, usually for restarting services.

Example:

```yaml
notify: restart nginx
```

---

### 9. How can variables be defined in Ansible?

**Answer:**

- In playbooks (`vars`)
- Inventory file
- `group_vars` and `host_vars`
- `vars_files`
- Extra variables (`-e`)

---

### 10. What are Ansible Facts?

**Answer:**
Facts are system-generated variables that store information such as:

- OS type
- IP address
- Memory

Example:

```yaml
{ { ansible_os_family } }
```

---

### 11. What are Roles in Ansible?

**Answer:**
Roles provide a **structured and reusable** way to organize playbooks.

Directory structure:

```

     ‚îú‚îÄ‚îÄ templates/

---

### 12. Difference between copy and template modules?

**Answer:**

- **copy:** Copies static files
- **template:** Copies dynamic files using **Jinja2**

---

### 13. What is Jinja2?

**Answer:**
Jinja2 is a **templating engine** used in Ansible to create dynamic configuration files using variables, loops, and conditions.

---

### 14. How do you use conditions in Ansible?

```
