# 🔵 Azure Interview Questions – System Admin & DevOps

## 🔹 Core Azure Basics (Must Know)

### 1️⃣ What is Azure?

**Answer:**
Microsoft Azure is a cloud platform providing **IaaS, PaaS, and SaaS** services for compute, storage, networking, security, and DevOps.

---

### 2️⃣ Azure IaaS vs PaaS?

**Answer:**

- **IaaS** → VM, VNet, Load Balancer (OS managed by us)
- **PaaS** → App Service, Azure SQL (OS managed by Azure)

---

### 3️⃣ Azure Regions & Availability Zones?

**Answer:**

- **Region** → Geographic location
- **Availability Zone** → Physically separate datacenters within a region (HA)

---

## 🔹 Azure Compute (System Admin Focus)

### 4️⃣ Types of Azure Virtual Machines?

**Answer:**

- General purpose (D-series)
- Compute optimized (F-series)
- Memory optimized (E-series)
- GPU (N-series)

---

### 5️⃣ VM Scale Set (VMSS)?

**Answer:**
Used to deploy and manage **identical VMs** with **auto-scaling**.

---

### 6️⃣ Difference: Azure VM vs App Service?

**Answer:**

- **VM** → Full OS control
- **App Service** → Platform-managed, no OS access

---

### 7️⃣ How do you secure Azure VM?

**Answer:**

- NSG rules
- Disable public IP
- Azure Bastion
- Patch management
- Disk encryption

---

### 8️⃣ Azure Bastion?

**Answer:**
Secure RDP/SSH access to VMs **without public IP**.

---

## 🔹 Azure Networking (Very Important)

### 9️⃣ What is VNet?

**Answer:**
A private network in Azure where resources communicate securely.

---

### 🔟 NSG vs Azure Firewall?

**Answer:**

| NSG             | Azure Firewall      |
| --------------- | ------------------- |
| Layer 4         | Layer 7             |
| Basic filtering | Advanced protection |
| Free            | Paid                |

---

### 1️⃣1️⃣ Load Balancer vs Application Gateway?

**Answer:**

- **Load Balancer** → Layer 4 (TCP/UDP)
- **App Gateway** → Layer 7 (HTTP/HTTPS, WAF)

---

### 1️⃣2️⃣ What is Private Endpoint?

**Answer:**
Provides private access to Azure services via **VNet**, no public exposure.

---

### 1️⃣3️⃣ Azure DNS vs Traffic Manager?

**Answer:**

- **Azure DNS** → Domain management
- **Traffic Manager** → Global traffic routing

---

## 🔹 Docker – Advanced Level (6+ Years)

### 1️⃣ Difference between Docker Image and Container?

**Answer:**

- **Image** → Read-only template (blueprint)
- **Container** → Running instance of an image
  One image can create multiple containers.

---

### 2️⃣ Dockerfile vs Docker Compose?

**Answer:**

- **Dockerfile** → Builds a single image
- **Docker Compose** → Runs multiple containers (microservices) using `docker-compose.yml`

---

### 3️⃣ What is multi-stage Docker build?

**Answer:**
Used to reduce image size by separating **build stage** and **runtime stage**.

```dockerfile
FROM node:18 AS build
RUN npm install

FROM node:18-alpine
COPY --from=build /app /app
```

---

### 4️⃣ How do you reduce Docker image size?

**Answer:**

- Use `alpine` images
- Multi-stage builds
- Remove unused packages
- Combine RUN commands
- `.dockerignore`

---

### 5️⃣ Docker networking types?

**Answer:**

- Bridge (default)
- Host
- None
- Overlay (Swarm / K8s)

---

### 6️⃣ Difference between CMD and ENTRYPOINT?

**Answer:**

- **CMD** → Default command, can be overridden
- **ENTRYPOINT** → Fixed command, arguments passed via CMD

---

### 7️⃣ How do containers communicate?

**Answer:**

- Same network → container name as hostname
- Different hosts → overlay network / K8s service

---

## 🔹 Kubernetes – Advanced (6+ Years)

### 8️⃣ What happens when a Pod crashes?

**Answer:**

- Kubelet restarts container
- If pod dies → ReplicaSet creates a new pod
- Controlled by **restartPolicy**

---

### 9️⃣ Pod vs Deployment?

**Answer:**

- **Pod** → Smallest unit
- **Deployment** → Manages replicas, rolling updates, rollback

---

### 🔟 What is a Service?

**Answer:**
Provides stable networking & load balancing for pods.

Types:

- ClusterIP
- NodePort
- LoadBalancer
- Headless

---

### 1️⃣1️⃣ ConfigMap vs Secret?

**Answer:**

- **ConfigMap** → Non-sensitive config
- **Secret** → Sensitive data (base64 encoded)

---

### 1️⃣2️⃣ Liveness vs Readiness Probe?

**Answer:**

- **Liveness** → Restart container if failed
- **Readiness** → Stop traffic if app not ready

---

### 1️⃣3️⃣ Rolling update vs Recreate?

**Answer:**

- **Rolling** → Zero downtime
- **Recreate** → Downtime allowed

---

### 1️⃣4️⃣ What is HPA?

**Horizontal Pod Autoscaler** scales pods based on:

- CPU
- Memory
- Custom metrics

---

### 1️⃣5️⃣ How does Kubernetes scheduling work?

**Answer:**
Scheduler checks:

- Node resources
- Taints & tolerations
- Node selectors / affinity

---

### 1️⃣6️⃣ What is etcd?

**Answer:**
Distributed key-value store storing **cluster state**.

---

### 1️⃣7️⃣ How do you secure Kubernetes?

**Answer:**

- RBAC

## 🔹 Basic Ansible Interview Questions

### 1. What is Ansible?

**Answer:**
Ansible is an **open-source automation tool** used for configuration management, application deployment, and orchestration.

---

### 2. Why is Ansible agentless?

**Answer:**
Ansible uses **SSH (for Linux)** and **WinRM (for Windows)** to communicate with servers, so no agent installation is required on managed nodes.

---

### 3. What are the main components of Ansible?

**Answer:**

- Inventory
- Playbook
- Modules
- Tasks
- Roles
- Handlers
- Facts

---

### 4. What is an Inventory file?

**Answer:**
An inventory file contains the **list of target servers** (IP addresses or hostnames) managed by Ansible.

Example:

```ini
[web]
192.168.1.10
192.168.1.11
```

---

### 5. What is a Playbook?

**Answer:**
A playbook is a **YAML file** that defines a set of tasks to be executed on managed hosts.

---

### 6. What is YAML?

**Answer:**
YAML stands for **Yet Another Markup Language**.
It is a human-readable data serialization format based on indentation.

---

## 🔹 Intermediate Ansible Interview Questions

### 7. What is the difference between a Module and a Task?

**Answer:**

- **Module:** A reusable unit that performs a specific action (copy, yum, service).
- **Task:** Executes a module with specific parameters.

---

### 8. What are Handlers?

**Answer:**
Handlers are special tasks that run **only when notified**, usually for restarting services.

Example:

```yaml
notify: restart nginx
```

---

### 9. How can variables be defined in Ansible?

**Answer:**

- In playbooks (`vars`)
- Inventory file
- `group_vars` and `host_vars`
- `vars_files`
- Extra variables (`-e`)

---

### 10. What are Ansible Facts?

**Answer:**
Facts are system-generated variables that store information such as:

- OS type
- IP address
- Memory

Example:

```yaml
{ { ansible_os_family } }
```

---

### 11. What are Roles in Ansible?

**Answer:**
Roles provide a **structured and reusable** way to organize playbooks.

Directory structure:

```
roles/
 └── web/
     ├── tasks/
     ├── handlers/
     ├── vars/
     ├── templates/
```

---

### 12. Difference between copy and template modules?

**Answer:**

- **copy:** Copies static files
- **template:** Copies dynamic files using **Jinja2**

---

### 13. What is Jinja2?

**Answer:**
Jinja2 is a **templating engine** used in Ansible to create dynamic configuration files using variables, loops, and conditions.

---

### 14. How do you use conditions in Ansible?

**Answer:**

```yaml
when: ansible_os_family == "RedHat"
```

---

### 15. Give an example of a loop.

**Answer:**

```yaml
loop:
  - nginx
  - git
```

---

## 🔹 Advanced Ansible Interview Questions

### 16. What is Ansible Vault?

**Answer:**
Ansible Vault is used to **encrypt sensitive data** like passwords and API keys.

Commands:

```bash
ansible-vault create secrets.yml
ansible-vault edit secrets.yml
```

---

### 17. What is Dynamic Inventory?

**Answer:**
Dynamic inventory automatically fetches host details from **cloud platforms** like AWS, Azure, or GCP at runtime.

---

### 18. What is Idempotency?

**Answer:**
Idempotency means running a playbook multiple times will **not change the system after the first successful run**.

---

### 19. Push vs Pull model?

**Answer:**

- **Ansible:** Push-based
- **Puppet / Chef:** Pull-based

---

### 20. What is the use of the serial keyword?

**Answer:**
Used for **rolling deployments**, executing tasks on a limited number of hosts at a time.

```yaml
serial: 2
```

---

### 21. What are Ad-hoc commands?

**Answer:**
One-line commands used for quick tasks without writing playbooks.

```bash
ansible all -m ping
```

---

### 22. How is error handling done in Ansible?

**Answer:**

```yaml
ignore_errors: yes
```

or

```yaml
failed_when:
```

---

### 23. What is Ansible Galaxy?

**Answer:**
Ansible Galaxy is a **repository of reusable community roles**.

---

### 24. What are Tags used for?

**Answer:**
Tags allow execution of **specific tasks or roles**.

```bash
ansible-playbook site.yml --tags nginx
```

---

### 25. Ansible vs Terraform?

**Answer:**

- **Ansible:** Configuration management & automation
- **Terraform:** Infrastructure provisioning

---
