# Q41. DNS is not resolving. How do you troubleshoot?

## Answer

If a hostname cannot be resolved into an IP address, troubleshoot DNS layer by layer.

---

## Step 1: Test IP Connectivity

```bash
ping 8.8.8.8
```

If this works, the network is fine.

---

## Step 2: Test DNS Resolution

```bash
ping google.com
```

If this fails but IP works, it's a DNS issue.

---

## Step 3: Check DNS Configuration

```bash
cat /etc/resolv.conf
```

Example

```text
nameserver 8.8.8.8
nameserver 1.1.1.1
```

---

## Step 4: Test DNS Server

```bash
dig google.com
```

or

```bash
nslookup google.com
```

---

## Step 5: Check Firewall

```bash
firewall-cmd --list-all
```

DNS uses:

```text
UDP 53
TCP 53
```

---

## Common Causes

- Wrong nameserver
- DNS server down
- Firewall blocking port 53
- Network issue
- Incorrect search domain

---

## Interview Tip

Always verify:

Network → DNS Configuration → DNS Server → Firewall

<a id="q42"></a>

# Q42. DNS resolution is very slow. What could be the reasons?

## Answer

Slow DNS increases application response time and SSH login delays.

---

## Common Causes

- Slow DNS server
- Unreachable primary DNS
- Reverse DNS timeout
- Incorrect search domain
- Firewall delay
- High network latency

---

## Verify

```bash
time nslookup google.com
```

or

```bash
time dig google.com
```

---

## Check SSH Configuration

```bash
grep UseDNS /etc/ssh/sshd_config
```

Recommended

```text
UseDNS no
```

Restart SSH

```bash
systemctl restart sshd
```

<a id="q44"></a>

# Q44. Why is /etc/resolv.conf overwritten automatically?

## Answer

Several services can regenerate `/etc/resolv.conf`.

---

## Common Services

- NetworkManager
- DHCP Client
- systemd-resolved
- cloud-init

---

## Verify

```bash
ls -l /etc/resolv.conf
```

Example

```text
/etc/resolv.conf -> ../run/systemd/resolve/stub-resolv.conf
```

---

## NetworkManager

```bash
nmcli connection show
```

---

## systemd-resolved

```bash
resolvectl status
```

---

## Interview Tip

Do not edit `/etc/resolv.conf` directly unless you know what service manages it.

<a id="q45"></a>

# Q45. Difference between /etc/hosts and /etc/resolv.conf?

## Answer

| File               | Purpose                                     |
| ------------------ | ------------------------------------------- |
| `/etc/hosts`       | Local hostname-to-IP mapping                |
| `/etc/resolv.conf` | Defines DNS servers for hostname resolution |

---

## Example

### /etc/hosts

```text
192.168.1.10 appserver
```

### /etc/resolv.conf

```text
nameserver 8.8.8.8
```

---

## Interview Tip

Linux checks `/etc/hosts` before querying DNS (depending on the system's `nsswitch.conf` configuration).

# Q46. What is Reverse DNS? How do you troubleshoot it?

## Answer

Reverse DNS converts an IP address into a hostname.

---

## Check

```bash
dig -x 8.8.8.8
```

or

```bash
host 8.8.8.8
```

---

## Common Problems

- Missing PTR record
- Incorrect reverse zone
- DNS server misconfiguration

---

<a id="q47"></a>

# Q47. What is a Search Domain?

## Answer

A search domain allows users to resolve short hostnames without typing the full domain name.

---

## Example

```text
search company.local
```

Typing

```text
server01
```

Automatically becomes

```text
server01.company.local
```

---

## Verify

```bash
cat /etc/resolv.conf
```

# Q48. How do you troubleshoot DNS cache issues?

## Answer

Cached records may return outdated IP addresses.

---

## Flush Cache

systemd-resolved

```bash
resolvectl flush-caches
```

---

## Restart Resolver

```bash
systemctl restart systemd-resolved
```

---

## Verify Cache Statistics

```bash
resolvectl statistics
```

# Q50. Server cannot resolve hostnames. How will you troubleshoot?

## Answer

Follow a structured approach:

### Step 1

Verify network connectivity.

```bash
ping 8.8.8.8
```

---

### Step 2

Verify DNS configuration.

```bash
cat /etc/resolv.conf
```

---

### Step 3

Check local hostname mapping.

```bash
cat /etc/hosts
```

---

### Step 4

Test DNS server.

```bash
dig google.com
```

---

### Step 5

Verify DNS service.

```bash
resolvectl status
```

or

```bash
systemctl status systemd-resolved
```

---

### Step 6

Check firewall.

Ensure UDP/TCP port 53 is allowed.

---

### Step 7

Review logs.

```bash
journalctl -u systemd-resolved
```

---

## Common Causes

- Wrong nameserver
- DNS server unavailable
- Firewall blocking port 53
- Incorrect search domain
- Broken `/etc/resolv.conf`
- Network connectivity issue

---

# ⭐ Top Interview Question

### Interviewer:

> "Users can ping IP addresses but cannot ping hostnames. What is your troubleshooting approach?"

### Best Answer

1. Verify IP connectivity (`ping 8.8.8.8`)
2. Check `/etc/resolv.conf`
3. Verify nameserver entries
4. Test with `dig` or `nslookup`
5. Check `/etc/hosts`
6. Verify `systemd-resolved` or `NetworkManager`
7. Ensure firewall allows TCP/UDP port 53
8. Review resolver logs
9. Test another DNS server
10. Verify reverse DNS if required
