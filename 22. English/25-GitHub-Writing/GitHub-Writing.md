# GitHub Writing

## Introduction

GitHub Writing means writing:

- README Files
- Project Documentation
- Installation Guides
- Technical Notes
- Learning Notes
- DevOps Documentation

in a professional and easy-to-understand way.

---

## Hindi Meaning

GitHub Writing का मतलब GitHub पर Professional Documentation लिखना है।

---

## Marathi Meaning

GitHub Writing म्हणजे GitHub वर व्यावसायिक Documentation लिहिणे.

---

# Why GitHub Writing is Important?

Good GitHub documentation helps:

✅ Recruiters understand your skills

✅ Team members understand your project

✅ Interviewers evaluate your communication

✅ Future you understand your own project

# Writing Rules

---

## Rule 1

Use simple English.

### Bad

The application facilitates infrastructure orchestration mechanisms.

### Good

The application automates infrastructure deployment.

---

## Rule 2

Write short sentences.

### Bad

This project is used for monitoring multiple systems and generating reports and alerts and dashboards.

### Good

This project monitors systems.

It generates alerts and reports.

---

## Rule 3

Use bullet points.

### Good

- Easy to use
- Lightweight
- Fast deployment

---

# Project Description Writing

---

## Formula

```text
What?

Why?

How?
```

---

## Example

### What?

This project automates server deployment.

### Why?

To reduce manual effort.

### How?

Using Terraform and AWS.

---

## Final Version

This project automates server deployment using Terraform and AWS. It reduces manual effort and improves consistency.

# Installation Section

Always write step-by-step instructions.

---

## Example

### Clone Repository

```bash
git clone https://github.com/user/project.git
```

### Move into Project

```bash
cd project
```

### Run Application

```bash
docker-compose up -d
```

---

# Usage Section

Explain how to use the project.

---

## Example

```text
1. Start the application.

2. Access the dashboard.

3. Configure alerts.

4. Monitor server health.
```

# Common GitHub Writing Mistakes

---

## Wrong

❌ This project is very useful.

### Better

✅ This project automates server deployment.

---

## Wrong

❌ Easy project.

### Better

✅ A simple project for learning Docker basics.

---

## Wrong

❌ Working project.

### Better

✅ A production-ready monitoring solution.
