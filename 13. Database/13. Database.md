# ðŸ—‚ï¸ **2ï¸âƒ£ Relational Databases (RDBMS)**

> Data stored in **tables**, connected by **relationships** (rows/columns).

*(Most RDBMS use SQL but here category focuses on structure)*

**1. MySQL**
Stores data in tables and supports relationships through primary/foreign keys.

**2. PostgreSQL**
Advanced relational engine with high reliability and complex relation handling.

**3. MariaDB**
MySQL fork with better performance and strong relational architecture.

**4. SQL Server**
RDBMS with strong transactional integrity and enterprise reliability.

**5. Oracle DB**
Robust RDBMS with strong security, clustering, and high availability for enterprises.

**6. IBM Db2**
Highly secure and scalable relational database for financial & enterprise systems.

# ðŸ—‚ï¸ **3ï¸âƒ£ Non-Relational Databases (NoSQL)**

> **Schema-less**, handles unstructured or semi-structured data, high scalability.

**1. MongoDB**
Document-based NoSQL DB storing JSON-like data; ideal for flexible schemas.

**2. Redis**
In-memory key-value NoSQL DB used for caching, sessions, and real-time operations.

**3. Cassandra**
Wide-column NoSQL DB built for extreme scalability and distributed workloads.

**4. DynamoDB (AWS)**
Serverless key-value NoSQL DB with millisecond latency and auto-scaling.

**5. CouchDB**
Document NoSQL DB focusing on replication and offline-first syncing.

**6. Elasticsearch**
Search + analytics engine storing JSON documents; used for logs and full-text search.

**7. InfluxDB**
Time-series NoSQL DB optimized for metrics, DevOps monitoring & IoT data.

**8. Neo4j**
Graph NoSQL DB storing nodes & relationships; used in recommendation systems.
