## 🔧 Basic Ubuntu VM Configuration

### 1. Set Hostname

This helps identify your VM in a multi-node setup.

```bash
sudo hostnamectl set-hostname vm1-ci-cd
```

Replace `vm1-ci-cd` with something meaningful (e.g., `k8s-master`, `jenkins-node`).

Check:

```bash
hostnamectl
```

---

### 2. Configure Static IP Address

By default, Ubuntu uses DHCP. For a lab, static IPs are better.

Edit Netplan config (usually in `/etc/netplan/01-netcfg.yaml`):

```bash

- Find your network interface name

You’ll see something like ens33, eth0, or enp0s3. Note the correct name.
# ip link


for check gateway IP

# ip route

```

```yaml
network:
  version: 2
  ethernets:
    ens33:
      dhcp4: no
      addresses:
        - 192.168.16.131/24 # your chosen static IP
      routes:
        - to: default
          via: 192.168.16.2 # <-- gateway from ip route
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]
```

Apply changes:

```bash
sudo chmod 600 /etc/netplan/01-netcfg.yaml
sudo netplan apply
ip addr show ens33
ip route
ping 8.8.8.8
```

Check IP:

```bash
ip addr show
```

### 3. Update & Upgrade Packages

```bash
sudo apt update && sudo apt upgrade -y
```

---

### 4. Install VMware Tools (for smooth integration)

```bash
sudo apt install open-vm-tools open-vm-tools-desktop -y
```

---

### 5. Create a User (if needed)

```bash
sudo adduser devops
sudo usermod -aG sudo devops
```

---

### 6. Enable SSH Access

So you can connect from your host:

```bash
sudo apt install openssh-server -y
sudo systemctl enable ssh
sudo systemctl start ssh
```

---

### 7. Verify Networking Between VMs

Ping other VMs by hostname or IP to ensure connectivity:

```bash
ping 192.168.1.101
```

---

### 8. Enable Essential Utilities

```bash
sudo apt install curl wget git vim -y
```
